Experimental Features (IMPORTANT)

In some circumstances, especially with quick redraws, you may experience a lot of flickering.

I'm working on a solution using the BeginRefresh() / EndRefresh() commands entirely handled by HGui which will keep tracks of the windows where it is active and where is not.

This is necessary because HGui and its layout system are heavily recursive so a handler for those mentioned function is necessary.

Right now this feature is implemented only on the gadget:set() method, since it is experimental you may experience malfunctions so you have the possibility to switch this feature OFF simply typing this code in your source:

HGui.RefreshBlockEnabled = False

 

Please report any issue you may find because with this feature a very smooth UI is really possible even with very complex interface.
