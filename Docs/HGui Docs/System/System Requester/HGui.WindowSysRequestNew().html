HGui.Window:SysRequestNew()

result = HGui.Window:SysRequest(params)

Create a system request window to show messages and let the user interact with the available options.
The requester can work in two modes 'blocking' and 'non-blocking', in both modes button's callbacks and a global callback function can be used to manage the specified buttons.
In 'blocking' mode the code flow will be blocked until an option is selected and the requester will be threated as a modal window. This mode will return the number of the selected option or -1 if the requester has been closed by its close gadget.
If used in ‘non-blocking’ mode the code flow will continue and when an event is detected the associated callback functions will be called.

INPUT

  • params : A table with the standard window option and the requester specific ones. Below there is a list of the additional fields supported by this method.
    • name : Unique name window
    • icon : Optional icon name/brush id
    • blocking : True or False depending if you want to block the code flow or not
    • text : Message to show in the requester body
    • wordwrap : Optional flag to set if the text should be word-wrapped (default is False)
    • buttons : The button you want to show in the requesters, this is a table with one subtable per button like this
                       {  { button_parameters }, 
                           ...                   
                           }                     
    • align : Text alignment, default = #HGUI_ALIGN_CENTER
    • buttonWeights : A table with the button weights (optional)
    • groupWeights : A table with the body & button weights (optional)
    • layout : Buttons alignment and arrangement, one of the following:
           #HGUI_VERTICAL_LAYOUT   
           #HGUI_HORIZONTAL_LAYOUT 
           #HGUI_GRID_LAYOUT       
    • callback : You can define your callback functions in the buttons parameters (like you do with standard buttons) or you can use a global callback that will be called with the following parameters:
      • Window object
      • Button object (empty table if the requester is closed)
      • Button number (-1 if the requester is closed)

OUTPUT

The output depends on the blocking flag.

BLOCKING = False 
This method returns the requester's window object immediately. The callback functions will be executed when an event is detected and will return the window object, the button object and the button number (-1 if the window was closed).

winObj = HGui.Window:SysRequestNew()
 

BLOCKING = True
This function returns the window object, the button object and the button number (-1 if the window was closed), also the callback function will be executed if specified.

winObj, btnObj, btnNum = HGui.Window:SysRequestNew()

 

NOTES
In any case the requester window will be closed and set free automatically.

Here are some default values used if you do not specify them:

params.size     = HGui.Theme.SysRequest.WindowSize
params.text     = ""
params.icon     = ""
params.align    = #HGUI_ALIGN_CENTER
params.buttons  = { { Caption = "OK" } }
params.layout   = #HGUI_VERTICAL_LAYOUT
params.callback = Function() EndFunction
params.blocking = True
params.wordwrap = False
