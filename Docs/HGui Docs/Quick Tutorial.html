Quick Tutorial

This is a very quick tutorial to let you start using HGui.

 

HOW TO INSTALL HGui and it's related modules

  1. Head your browser at my GitHub page → https://github.com/Allanon71/Hollywood-MAL-Libraries
  2. Click on the green button “Code” and a menu will open
  3. On your LOCAL tab press the “Download ZIP” button to download the entire repository on your local machine.
  4. Once the download ends open the archive and uncompress the contents where you prefer
  5. Open the file +Includes.hws and edit the line Const #INC_PATH = "", you have to specify the absolute location where you have uncompressed the archive, for example it could be something like this: 
    C:/MyHollywoodStuff/Libs/AllanonLibs/
  6. Save the edited file
  7. You are ready!

 

INCLUDING HGui

To work with HGui the first step in your code should to include the library into your source, if the previous step was correct (installation) it's enough to add these two lines into your source code:

@INCLUDE "<path-where-you-have-extracted-the-archive>/+Includes.hws"
@INCLUDE #INC_HGUI

Inside the file +Includes.hws you will find some constants you can use to include my libraries into your projects, even without using HGui. For example, if you would want to use only the Date & Time library you'd only need to:

@INCLUDE "<path-where-you-have-extracted-the-archive>/+Includes.hws"
@INCLUDE #INC_DATETIME

 

HELLO WORLD WITH HGui

Tutorial T304 - Hello World.hws

NOTE
This example expects that you have not changed the examples directory, they should be in <path-where-you-have-extracted-the-archive>/Examples/HGui that's why at the beginning of the code we are referring to the +Includes.hws file two directories up.

Let's start including HGui

@INCLUDE "../../+Includes.hws"
@INCLUDE #INC_HGUI

Now we are going to create a window to contain our ‘Hello World’ text, you can see that we are specifying a few arguments to:
- Setup the window's title
- Setup the window's name, it is used internally
- Specify how the window should be positioned

Local myWin = HGui.Window:new(
   { title = "Hello World!",
     name  = "myWindow",
     position   = { x = #CENTER, y = #CENTER }
     })

The following  command it's not really needed because we are going to add only a simple label gadget, but it's a good practice to switch off auto rendering while building GUIs because if not disabled, everytime you add a new gadget, a window refresh will be issued causing flickering which is not so nice to see.

HGui.SetAutoRendering(False)

Now it's time to create our label gadget that will be used to show the “Hello World!” string:

Local myLabel = HGui.Gadget:new(#HGUI_LABEL_GADGET, { caption = "Hello World!" })

Just creating a gadget it's not enough, we need to tell HGui where we have to place it, this task is accomplished using the :layoutSet() method, as you can see we are using the method with the root gadget of our window. The root gadget is a special gadget created for you when a window is created, it is the root of every gadget you are going to place in the window's gui.

mywin.rootgadget:layoutSet({ gadgets = { myLabel } }, True )

Now let's enable the auto rendering feature:

HGui.SetAutoRendering(True)

To be sure that all has been rendered correctly let's force a window refresh:

myWin:Render()

Is it simple enough? :D

Last step is to enter a neverending loop because HGui is event driven and we do not need to listen manually for events:

Repeat
  WaitEvent()
Forever

Open your task manager and see how CPU time this program takes, it should be very low.
