Project Description

INTRODUCTION

HGui is one of the biggest project I worked on and its development started back in the 2011.

The very first attempt was called scuiLib but after a few years I realized how to properly code such a project using OOP to simplify the code required to build user interfaces.

I've to admit that MUI inspired me a lot in this adventure and now, even if the project it's still incomplete, I can say that it's very robust, flexible and very usable. I cannot say this was easy… but I enjoyed (and still enjoying) the development of HGui. 

I've used it to produce many applications, even very heavy ones in terms of interface complexity, without major issues.

Of course all can be made better and I will do it as soon as me or users will need new features that will make HGui even better.

I've to be honest: I'm very proud of this work :D

 

FEATURES

  • Object Oriented Programming, you will code your app GUIs using objects and methods
  • Flexible and multi-layered skin system, you can create even complex themes or very simple (and light) ones, the skin system offers you a fully featured subsystem.
  • Easy of use, at least for me :)
  • Fast, especially when using light skins and/or using the cache system (enabled by default)
  • Scalable (tuning the cache system and the theme you can balance the cpu consumption and the memory usage)
  • Pure Hollywood code : this means that HGui does not depend on any system specific command/library, all developed apps using HGui can be compiled for any system supported by Hollywood (Android needs some attention).
    Your app GUI will be the same on every platform, exactly the same.
    The only requirements are a few support includes (written in pure Hollywood by me) available with HGui.
  • WINDOWS
    • Every HGui window supports the following events:
      OnClose, OnMove, OnActivate, OnDeactivate, OnHide, OnShow, OnSize
    • A lot of customization is available when creating new windows
    • Automatic control of the window’s order (window's stack)
    • Methods to control/edit on the fly the created windows
    • Various window's background types supported
    • Special methods to quickly build:
      • Progress indicators
      • System requesters
      • Text boxes
      • Wait windows
      • Soft Keypads
      • Soft Keyboards
      • Input boxes
      • Message boxes
    • Custom window’s menu (not the ones available with Hollywood but a custom one that allow you to include any gadget type in it).
    • Utility functions to find window objects by name, by Hollywood ID, and to check if a window exists or not.
    • Ability to attach hotkeys to any window to execute specific tasks when a key press occurs.
  • GADGET CLASSES
    • Box
    • Vertical Divider
    • Horizontal Divider
    • Label
    • Group (Generic, Radio, Check)
    • Button
    • Switch
    • Scrollbar (Vertical/Horizontal)
    • Slider (Vertical/Horizontal)
    • Progress Bar
    • Knob
    • Text (for user input)
    • Listview
    • Treeview
    • Drop down
    • Image
    • Virtual Area
    • Canvas
  • GADGET SYSTEM
    • Ability to switch the cache system on or off,  when enabled each gadget state will be cached, a lot faster but more memory is required. When disabled each gadget state will be rendered on the fly, slower but with a much lower memory footprint.
    • Each gadget can have its own skin.
    • Each gadget class can have its own skin.
    • Support for icons inside gadgets with custom alignment and text adjustment
    • Flying tips for active gadgets
    • Hotkeys can be defined for each active gadget
    • You can attach sounds to some actions (OnOver, OnOut, OnPushed, OnDown)
    • Supported actions (some gadget classes may have additional actions):
      • OnPushed : Triggered when the left mouse button released (always after an OnDown event)
      • OnDown : Triggered when the left mouse button is pressed
      • OnOver : Triggered when the mouse enters a gadget's area
      • OnOut : Triggered when the mouse exits a gadget's area
      • OnDrop : Triggere when something is dropped inside a gadget's area
    • Functions to search buttons by their names
    • Methods to control and modify on the fly existing gadgets
    • Ability to attach a right-mouse action or a pop up menu on each active gadget
  • LAYOUT SYSTEM
    Gadgets can be arranged using the famous MUI library concept: you define some gadgets that will be grouped inside another gadget, this way your GUI is represented by a tree where the tree root is a special gadget called rootGadget, this special gadget is created for you automatically when a window is created.
    This system supports 3 layouts:
    • Vertical
    • Horizontal
    • Grid
  • When you define child gadgets you can define the following properties:
    • Gadget weights (percentual variable, percentual fixed, pixel variable, pixel fixed)
    • Borders (top, bottom, left, right) from the parent gadget
    • Gap between gadget childs in pixel

 

WHAT’S MISSING/NEEDS MORE LOVE

As I said HGui is still missing some core features like the ones listed below:

  • Keyboard navigation system (very high priority)
  • Gadget’s drag & drop (drag one or more gadgets over another one) (low priority)
  • Tabs class (high priority)
  • Spin class (normal priority)
  • Better documentation (yes… this one isn't enough, I will improve it if there is enough interest from users)
  • Tutorials (Some basic examples are provided but they do not cover everything HGui can offer)
  • Global theme management functions are not complete (high priority)
  • Support for a way to separate Gui definitions from the code (low priority)
